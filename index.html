About Me

I am a student at UC San Diego getting my BS in Computer Science. Something I've learned over my internships and time in school is this: the people are important. Good teams are what makes good software. My goal is not to just be a good software engineer. I want to be an effective colleague who contributes positively to the team's success.

Through my past internships, I have gained experience in both .NET and back-end Java development. I am a fast learner and a passionate worker. Facing problems in unfamiliar territory is a fun and exciting task for me. If there is a technology I am unfamiliar with, I will take the time to learn it and apply it to the problems I face.

Waitless

Class projects, although fun, can often fail to simulate what software development is like in the real world. This UCSD class gave my team a simple task: create a visitor management web application. The catch was this: we had to inherit code from a previous class. After 10 weeks of development, my team delivered an improved application with newly implemented features.

Project Info

  • Client: UCSD Course
  • Date: June 2017
  • Service: Web Development
  • Framework: Node.js

Challenges

  1. Inherited large codebase with thousands of lines, much of which didn't do anything
  2. Most of the features were buggy and had to be fixed
  3. Existing tests were mostly broken and needed to be rewritten

Features Added

New Admin Dashboard to keep track of visitor info
Fixed bugs in the appointment scheduler
Integrated appointment system with Facebook messaging

Deployment Pipeline

Developed an fully automated deployment pipline. With every push to Git, this pipeline runs several tools to test code and ensure code quality.
